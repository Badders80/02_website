## Known Issues

### Cloud Function Authentication

**Issue:** Organization policy blocking `--allow-unauthenticated` flag

The Cloud Function is deployed but returns 403 for unauthenticated requests due to GCP organization policy restrictions on the work account (`alex@evolutionstables.nz`).

**Root Cause:**
- GCP organization policy prevents granting `allUsers` access
- Build succeeds, but deployment fails at IAM policy check

**Solutions:**

1. **Use WIF Authentication** (Recommended)
   - Frontend already has WIF infrastructure
   - Next.js API routes (`/api/applications/submit`) already send GCP identity tokens
   - Function will work when called through the frontend

2. **Request Org Policy Exception**
   - Contact GCP org admin
   - Request exception for this specific function
   - Or allow `allUsers` for Cloud Functions in this project

3. **Grant Vercel OIDC Service Account Access**
   - Get Vercel OIDC service account email
   - Grant `roles/cloudfunctions.invoker` role
   - Update function to accept requests from this SA only

**Workaround:**
The function is designed to work with authentication. The Next.js API routes in `/api/applications/` already include GCP identity token exchange via WIF. Test the full flow through the website.
