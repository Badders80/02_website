# Content

**Purpose:** JSON content files that drive website pages — copy, structure, and metadata.

---

## Why JSON Content?

- **Separation of concerns** — Content lives in JSON, presentation in React
- **Easy updates** — Change copy without touching code
- **Future CMS** — JSON can be generated by a headless CMS later

## Files

| File | Drives | Status |
|------|--------|--------|
| `homepage.json` | Homepage sections, hero text, CTA | ⬜ Not created |
| `press.json` | Press releases, media kit | ⬜ Not created |

## Format

Each JSON file maps to a page. Structure:
```json
{
  "meta": { "title": "", "description": "" },
  "sections": [
    { "type": "hero", "heading": "", "body": "", "cta": { "label": "", "href": "" } }
  ]
}
```
